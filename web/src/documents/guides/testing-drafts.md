# Draft Review UI - Testing Guide

## Overview

Phase 4 Draft Review UI has been implemented. This allows admins to review, edit, approve, or reject daily posts generated by the digest service.

## What Was Built

### Backend API (Express)
- **Routes**: `/api/drafts`
  - `GET /api/drafts` - List all drafts with filters (status, targetDate, timeSlot)
  - `GET /api/drafts/:id` - Get single draft with full details
  - `PATCH /api/drafts/:id` - Update draft content
  - `POST /api/drafts/:id/approve` - Approve draft (DRAFT → APPROVED)
  - `POST /api/drafts/:id/reject` - Reject draft (DRAFT → REJECTED) with reason

- **Files Created**:
  - `server/src/schemas/draft.schema.ts` - Zod validation schemas
  - `server/src/services/draft.service.ts` - Business logic
  - `server/src/controllers/draft.controller.ts` - Request handlers
  - `server/src/routes/draft.routes.ts` - Route definitions
  - `server/src/routes/index.ts` - Updated to include draft routes

### Frontend UI (React + TailwindCSS)
- **DraftsPage** (`/drafts`) - Main page showing all drafts in a table
  - Filter by status (DRAFT/APPROVED/REJECTED/POSTED)
  - Filter by target date
  - View content preview
  - Edit/View button for each draft
  - Stats cards showing count by status

- **DraftEditor Component** - Modal for editing drafts
  - Tabs: Preview, Hook, Bullets, OCVN Take, CTA, Hashtags
  - Character counters for each section
  - Save changes button
  - Approve/Reject buttons (for DRAFT status only)
  - Source links display
  - Rejection dialog with reason input

- **Files Created**:
  - `web/src/pages/DraftsPage.tsx` - Main drafts management page
  - `web/src/components/DraftEditor.tsx` - Draft editing modal
  - `web/src/types/api.ts` - Updated with Draft types
  - `web/src/lib/api-client.ts` - Updated with draft API methods
  - `web/src/App.tsx` - Updated with `/drafts` route
  - `web/src/pages/SourcesPage.tsx` - Updated header with navigation

## Prerequisites

### 1. Backend Setup
Ensure server is running with all migrations applied:

```bash
cd server
npm run prisma:migrate  # If not already done
npm run dev
```

Server should be running at `http://localhost:3001`

### 2. Frontend Setup
Ensure web app is running:

```bash
cd web
npm run dev
```

Web app should be running at `http://localhost:5173`

### 3. Generate Test Drafts
You need at least some drafts in the database. To generate them:

**Option A - Automatic (if pipeline is running):**
Wait for the digest job to run daily at 00:30, or trigger manually:

```bash
curl -X POST http://localhost:3001/api/admin/digest/trigger \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json"
```

**Option B - Check existing drafts:**
```sql
SELECT id, target_date, time_slot, status FROM daily_posts;
```

If no drafts exist, run the full pipeline first:
1. Add RSS sources (via UI or API)
2. Wait for ingest/extraction/filtering/AI jobs to run
3. Trigger digest generation

### 1. Login to Web UI

1. Navigate to `http://localhost:5173`
2. Login with admin credentials:
   - Email: `admin@opencampus.vn`
   - Password: `admin123` (or your configured password)

### 2. Navigate to Drafts Page

1. After login, you should be redirected to `/drafts` (new default)
2. Or click "Bài viết" in the navigation header
3. You should see:
   - A table listing all daily posts
   - Filters for status and date
   - Stats cards at the bottom

### 3. Test Filters

**Filter by Status:**
1. Select "📝 Nháp" (DRAFT) - should show only draft posts
2. Select "✅ Đã duyệt" (APPROVED) - should show only approved posts
3. Select "Tất cả" (ALL) - should show all posts

**Filter by Date:**
1. Pick a date from the date picker
2. Table should update to show only posts for that date

### 4. Test Draft Viewing

1. Click "👁️ Xem" or "✏️ Chỉnh sửa" on any draft row
2. Draft editor modal should open
3. Verify:
   - Header shows target date and time slot
   - Preview tab shows full content
   - Source links are displayed at bottom

### 5. Test Draft Editing (DRAFT status only)

**Edit Hook:**
1. Click "📢 Hook" tab
2. Edit the hook text
3. Click "💾 Lưu thay đổi"
4. Should see "✅ Đã lưu thay đổi" alert
5. Close modal and reopen - changes should persist

**Edit Bullets:**
1. Click "📝 Bullets" tab
2. Modify bullet points
3. Character counter should update
4. Save changes

**Edit OCVN Take:**
1. Click "💼 OCVN Take" tab
2. Edit the OCVN perspective text
3. Save changes

**Edit CTA:**
1. Click "📣 CTA" tab
2. Edit the call-to-action text
3. Save changes

**Edit Hashtags:**
1. Click "#️⃣ Hashtags" tab
2. Add/remove hashtags (space-separated, with # prefix)
3. Hashtag counter should update
4. Save changes

**Verify Changes:**
1. Switch to "👁️ Xem trước" (Preview) tab
2. Should show updated content combining all edited sections

### 6. Test Draft Approval

1. Open a draft with status DRAFT
2. Make any edits (optional)
3. Click "✅ Duyệt bài" button
4. Confirm in the dialog
5. Should see "✅ Đã duyệt bài viết" alert
6. Modal closes automatically
7. Draft status should change to "✅ Đã duyệt" in the table

**Validation:**
- Cannot approve a draft with empty content
- Only DRAFT status posts show approve button

### 7. Test Draft Rejection

1. Open a draft with status DRAFT
2. Click "❌ Từ chối" button
3. Dialog should appear asking for rejection reason
4. Enter a reason (e.g., "Nội dung không phù hợp")
5. Click "Xác nhận từ chối"
6. Should see "✅ Đã từ chối bài viết" alert
7. Modal closes automatically
8. Draft status should change to "❌ Đã từ chối" in the table

**Validation:**
- Rejection reason is required
- Only DRAFT status posts show reject button

### 8. Test Read-Only Mode

1. Open a draft with status APPROVED, REJECTED, or POSTED
2. Verify:
   - Edit tabs still accessible
   - Can view all content
   - **No approve/reject buttons visible**
   - Save button should still work (for notes/corrections)

### 9. Test Navigation

1. Click "Nguồn RSS" in header
2. Should navigate to `/sources` (Sources page)
3. Click "Bài viết" in header
4. Should navigate back to `/drafts` (Drafts page)

## Troubleshooting

### No Drafts Showing

**Problem**: Drafts page is empty

**Solutions**:
1. Check if digest job has run:
   ```sql
   SELECT COUNT(*) FROM daily_posts;
   ```

2. Manually trigger digest generation:
   ```bash
   curl -X POST http://localhost:3001/api/admin/digest/trigger \
     -H "Authorization: Bearer YOUR_JWT_TOKEN"
   ```

3. Ensure AI pipeline has completed items:
   ```sql
   SELECT COUNT(*) FROM items WHERE status = 'AI_STAGE_B_DONE';
   ```
   Need at least 6 items to generate drafts

### Cannot Edit Draft

**Problem**: Edit button doesn't open modal or save fails

**Solutions**:
1. Check browser console for errors
2. Verify JWT token is valid (logout and login again)
3. Check server logs for API errors
4. Verify draft status is DRAFT (only drafts can be edited)

### Approve/Reject Buttons Missing

**Expected Behavior**: These buttons only show for drafts with status DRAFT

If draft status is APPROVED, REJECTED, or POSTED, buttons are intentionally hidden.

### Changes Not Saved

**Problem**: Edits revert after closing modal

**Solutions**:
1. Make sure you clicked "💾 Lưu thay đổi" button
2. Wait for "✅ Đã lưu thay đổi" alert before closing
3. Check network tab for 401/403 errors (auth issue)
4. Verify backend is running and accessible

### Vietnamese Not Displaying Correctly

**Problem**: Vietnamese characters show as � or boxes

**Solutions**:
1. Ensure database uses UTF-8 encoding
2. Check Prisma connection string has `?schema=public` but no charset override
3. Verify browser encoding is set to UTF-8

## API Reference

### Endpoints

**GET /api/drafts**
- Query params: `status`, `targetDate`, `timeSlot`
- Returns: Array of DailyPost objects with populated relations

**GET /api/drafts/:id**
- Returns: Single DailyPost with full details

**PATCH /api/drafts/:id**
- Body: `{ hookText?, bulletsText?, ocvnTakeText?, ctaText?, hashtags? }`
- Returns: Updated DailyPost

**POST /api/drafts/:id/approve**
- Body: `{}`
- Returns: Updated DailyPost with status APPROVED

**POST /api/drafts/:id/reject**
- Body: `{ rejectionReason: string }`
- Returns: Updated DailyPost with status REJECTED

All endpoints require JWT authentication via `Authorization: Bearer <token>` header.
