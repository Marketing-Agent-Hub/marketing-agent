# Glossary

## Domain Terms

### Brand
The core business unit in the system. Represents a product, company, or marketing client. All content, strategies, and AI processing is organized around brands. A brand lives inside a Workspace.

### BrandProfile
An AI-generated structured document that captures a brand's identity: target audience, value propositions, tone guidelines, business goals, and messaging angles. Generated automatically from an onboarding transcript. Required before strategy generation can run.

### BrandSource
A many-to-many junction between a Brand and a Source. Each `BrandSource` can override the fetch interval and has its own `lastFetchedAt` tracker. Represents "Brand X is subscribed to Source Y."

### ContentBrief
The creative brief for a single social media post. Contains: title, objective, key angle, call-to-action, and asset direction. Generated by AI from a strategy slot, optionally grounded in a TrendSignal. The brief defines *what* to say; the draft defines *how* to say it.

### ContentDraft
The actual text of a social media post. Contains: hook, body, CTA, and hashtags. Generated by AI from a ContentBrief. Goes through a human review/approval workflow before being scheduled for publication.

### ContentPillar
A thematic topic cluster that a brand's content strategy is organized around (e.g., "Product Education", "Industry Insights", "Customer Success"). Each StrategySlot is associated with a pillar.

### FilterProfile
Per-brand configuration for the content filtering step. Can be `PASS_THROUGH` (all items pass) or `AI_EMBEDDING` (items must exceed a cosine similarity threshold with the brand's vector profile). Controls which news items are considered relevant to a brand.

### GenerationRun
An audit record of every AI inference call made for marketing workflows. Stores: workflow name, model, prompt version, token counts, input snapshot, and output snapshot. Used for debugging AI quality and tracking token costs.

### IngestJob
A background job that runs every 15 minutes. Fetches RSS/web feeds from all enabled sources and saves new articles as `Item` records with status `NEW`.

### Item
A single article or news post from a source. Has a strict status lifecycle: `NEW â†’ EXTRACTED â†’ READY_FOR_AI â†’ AI_STAGE_A_DONE â†’ AI_STAGE_B_DONE` (or `FILTERED_OUT` at various points). The central entity of the content intelligence pipeline.

### OnboardingSession
A structured conversation transcript between the frontend (posing questions) and a brand admin (answering). Used to collect brand information for AI-driven BrandProfile generation. Comprises a list of `{role, content, timestamp}` messages.

### OpenRouter
A third-party AI inference gateway that provides a single OpenAI-compatible API to access multiple AI models (GPT-4o, Claude, Llama, etc.). Used as the single AI backend for all inference calls.

### PendingSource
A proposed new RSS source discovered by the SourceDiscovery pipeline. Requires admin review (approve or reject) before becoming an active `Source`. Contains AI-scored metadata (trustScore, topicTags, suggestedDenyKeywords).

### PublishJob
A scheduled task to publish a specific `ContentDraft` to a social platform at a given time. Created when an APPROVED draft is scheduled. Executed by the PublishSchedulerJob every 5 minutes.

### Source
An RSS feed or web page to be periodically monitored. Has a `trustScore`, topic tags, deny keywords, and fetch interval. Can be enabled or disabled. Sources are global (shared) or brand-specific (via BrandSource).

### StrategyPlan
A time-bound content calendar. Contains weekly themes, a posting cadence config, and a set of StrategySlots. Exists in `DRAFT`, `ACTIVE`, `SUPERSEDED`, or `ARCHIVED` status. Only one strategy can be `ACTIVE` per brand at a time.

### StrategySlot
A single planned post within a StrategyPlan. Specifies: channel (platform), scheduled date/time, content pillar, and funnel stage. Advances through: `PLANNED â†’ BRIEF_READY â†’ DRAFT_READY â†’ APPROVED/SKIPPED`.

### TrendSignal
A high-importance, AI-processed news item that is surfaced as a "current trend" for use in marketing content generation. Created from `AI_STAGE_B_DONE` items when their importance score exceeds a threshold.

### BrandTrendMatch
A link between a TrendSignal and a Brand with a `relevanceScore`. Computed by comparing the cosine similarity between the trend's content embedding and the brand's topic profile.

### VectorProfile
A pre-computed embedding vector (float array) representing a brand's content interests. Stored in `FilterProfile.vectorProfile` as JSON. Used for AI_EMBEDDING mode filtering â€” items are embedded and compared against this vector.

### Workspace
The top-level multi-tenant container. Equivalent to an "account" or "organization." Users are members of workspaces with role-based access. All brands belong to a workspace.

## Technical Terms

### AI Stage A
The first AI processing step in the content pipeline. Uses a lightweight model (gpt-4o-mini by default) to classify an item: determines if content is relevant (`isAllowed`), assigns topic tags, importance score (0â€“100), and a one-line summary.

### AI Stage B
The second AI processing step. Uses a larger model (gpt-4o by default) to write a complete Vietnamese Facebook post from the article content. Produces `fullArticle` â€” a ready-to-copy social post with specific formatting conventions.

### asyncHandler
A utility wrapper at `src/lib/async-handler.ts` that wraps async Express route handlers in a try-catch that passes errors to the next middleware. Prevents unhandled promise rejection crashes.

### callAIWorkflow
A shared utility at `src/shared/marketing/ai-workflow.ts` that wraps AI API calls with: a `GenerationRun` audit record, input/output snapshotting, token counting, and error status tracking. Used by all marketing-domain AI calls.

### ContentHash
An MD5 or SHA hash of an article's core content, computed during ingestion. Used to detect duplicate entries of the same article (from different feeds or dates) and to cache AI Stage B results.

### DenyKeywords
A list of strings (stored per-source and globally) that cause an item to be filtered out if they appear in the title, snippet, or extracted content. Example: `["price", "trading", "pump"]` â€” market/trading content is globally banned.

### FunnelStage
A content marketing concept indicating where in the buyer journey a piece of content targets: `Awareness`, `Consideration`, or `Decision`. Assigned per StrategySlot.

### Internal Auth
Authentication for system administrators using credentials from environment variables (`ADMIN_EMAIL`, `ADMIN_PASSWORD_HASH`). Required for all `/api/internal/*` endpoints.

### Job Monitoring
A wrapper pattern (`src/lib/job-monitoring.ts`) that records metrics and logs for background job executions: start time, duration, success/failure, and error details.

### Plugin Registry
A map from `SourceType` enum values to plugin instances (RSS, WebScraper). Implements the Strategy pattern â€” ingest logic is identical regardless of source type; the plugin handles type-specific fetch and parse logic.

### Product Auth
Authentication for product users (brand managers, editors) using JWTs issued by the product auth service. Required for all `/api/*` product endpoints.

### PromptVersion
A semantic version string (e.g., `"1.0.0"`) that identifies which version of a prompt was used in an AI call. Stored in `GenerationRun` records. Allows correlation of output quality with specific prompt changes.

### requireBrandAccess(role)
Express middleware that verifies the authenticated user has at least the specified role for the brand being accessed. Resolves the brand's workspace and checks membership.

### requireWorkspaceAccess(role)
Express middleware that verifies the authenticated user has at least the specified role in the specified workspace.

### StubConnector
A fake implementation of the `SocialConnector` interface that simulates a successful social media publish. Returns a fake `externalPostId` without making any real API calls. Used for all platforms during development.

### Telemetry
The system's observability stack based on OpenTelemetry. Auto-instruments HTTP requests, DNS, and Node.js runtime. Exports metrics to Prometheus (via Prometheus exporter on a separate port) and traces to the database via `PerformanceTrace` records.

## Naming Conventions

| Convention | Example | Meaning |
|---|---|---|
| `*.service.ts` | `brand.service.ts` | Business logic layer |
| `*.controller.ts` | `brand.controller.ts` | HTTP request/response handling |
| `*.routes.ts` | `brand.routes.ts` | Express route definitions |
| `*.schema.ts` | `source.schema.ts` | Zod validation schemas for request bodies |
| `*.job.ts` | `ingest.job.ts` | Background job scheduler wrappers |
| `*.plugin.ts` | `rss.plugin.ts` | Source fetch/parse implementations |
| `*.repository.ts` | `pending-source.repository.ts` | Data access layer abstraction |
| `v2User` | `req.v2User` | Product JWT payload (named v2 to distinguish from legacy) |
| `internalUser` | `req.internalUser` | Internal admin JWT payload |
| Stage A / Stage B | `ai-stage-a` | The two sequential AI processing steps in content pipeline |
