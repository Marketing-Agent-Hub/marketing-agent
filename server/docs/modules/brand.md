# Module: Brand

## Purpose

Manages brand entities — the core business unit around which all marketing activity is organized. A brand represents a product or company being marketed within a Workspace.

## Key Files

| File | Role |
|---|---|
| `brand.service.ts` | CRUD for Brand records |
| `brand-analysis.service.ts` | AI-driven analysis of onboarding transcript → BrandProfile |
| `brand.controller.ts` | HTTP request handlers |
| `brand.routes.ts` | Route definitions |

## Responsibilities

### Brand CRUD
- **Create**: Associates a new brand with a workspace. Initial status: `ACTIVE`.
- **Get**: Returns brand with profile, pillars, and social account summary.
- **Update**: Allows updating name, websiteUrl, industry, timezone, defaultLanguage, status.
- **List**: Returns all brands in a workspace.

### Brand Knowledge Documents
- Brands can have attached reference documents (company FAQ, product sheets, tone guidelines) that the AI can use during content generation.
- Stored in `BrandKnowledgeDocument` table with `content`, `sourceUrl`, `docType`, `title`.
- Added via `POST /api/brands/:brandId/knowledge-documents`.

### Brand Analysis (`brand-analysis.service.ts`)
This is the AI engine that converts an onboarding conversation transcript into a structured `BrandProfile`. This service is called asynchronously after `OnboardingSession.complete()`.

**BrandProfile fields generated by AI:**
- `summary` — One paragraph describing the brand
- `targetAudience` — JSON object with demographics, psychographics
- `valueProps` — JSON array of key value propositions
- `toneGuidelines` — JSON object describing writing style, do's and don'ts
- `businessGoals` — JSON array of marketing objectives
- `messagingAngles` — JSON array of key content angles

The AI is given the full onboarding transcript and tasked with extracting or inferring all of the above fields. The model used is `marketing.models.businessAnalysis` (default: `gpt-4o`).

## Interactions With Other Modules

- **Workspace module**: A Brand always belongs to a Workspace. `workspaceId` is a required foreign key.
- **Onboarding module**: Completion of an onboarding session triggers `brand-analysis.service`.
- **Strategy module**: Strategy generation requires a brand profile to exist.
- **Content Intelligence**: Sources are associated with brands via `BrandSource`. Items are tagged with `brandId`.

## Brand Status

| Status | Meaning |
|---|---|
| `ACTIVE` | Brand is operational; jobs process its content |
| `PAUSED` | Temporarily inactive; jobs may skip it |
| `ARCHIVED` | Soft-deleted; shown in history only |
